# 개발 블로그 만들기

개인적으로 개발자에게 본인만의 블로그를 만드는 것은 꽤 도움이 많이 된다고 생각합니다.

### 본인의 개발 블로그를 만들었을 때 장점

> 1. 개발하다가 막혔을 때 공부한 것들을 기록하고 다시 찾아 볼 수 있다.
> 2. 취업할 때 플러스 요소가 될 수 있다. **(꾸준하게 관리했을 경우)**

<br />

본인만의 개발 블로그를 가졌을 때의 장점을 알았으니 이제 블로그를 만드는 방법을 알아보도록 하겠습니다.

## 1. 블로그 템플릿 선정

디자인과 프론트엔드 개발에 자신이 있으신 분들은 본인이 원하는 기술과 디자인으로 개발하시면 가장 좋습니다.  
하지만 본인이 직접 만들기 귀찮거나 저처럼 디자인이나 프론트엔드 개발에 자신이 없으신 분들은 미리 만들어져있는 오픈소스 블로그 템플릿을 이용하면 편하게 본인만의 개발 블로그를 만들 수 있습니다.

무료 템플릿을 알아보는 방법은 여러가지가 있겠지만 제가 제일 추천하는 방법은 [jamstackthemes.dev](https://jamstackthemes.dev/) 사이트에서 원하는 테마를 찾는 것을 추천드립니다.

<img
  src="/static/images/make-dev-blog/jamstack-themes.png"
  alt="jamstackthemes"

/>

많은 무료 템플릿 사이트 중에서 jamstackthemes를 추천하는 이유는 다음과 같습니다.

> 1. 원하는 SSG(Static Site Generaters)를 선택가능
> 2. 무료 테마를 커스텀하기 쉽게 본인에게 자신있는 css framework 선택가능

하지만 `Gatsby`와 `Jekyll` 테마가 가장 많고 나머지 SSG 테마가 부족하다는 단점이 있습니다.

<br />

- 제가 선택한 테마는 Next.js와 tailwind CSS로 만들어진 [tailwind-nextjs-starter-blog](https://github.com/timlrx/tailwind-nextjs-starter-blog)테마를 선택했습니다.  
  ruby 보다는 javascript가 익숙해서 Jekyll은 선택하지 않았습니다.

## 2. 배포 플랫폼 선정

프론트엔드 배포 플랫폼에는 대표적으로 다음과 같은 플랫폼들이 있습니다.

> 1. AWS Amplify
> 2. Netlify
> 3. Vercel
> 4. GitHub Pages

Jekyll 테마는 GitHub Pages에 배포하는 것이 편하고, Next.js는 Vercel에 배포하는 것이 더 편합니다.  
저는 Next.js 테마를 선택하였으므로 Vercel을 이용해서 배포하도록 하겠습니다.

## 3. Vercel을 이용해서 배포하기

1. [Vercel 홈페이지](https://vercel.com)에 접속해서 깃허브 계정을 연동하여 로그인 해 줍니다.
2. GitHub에 선택하신 테마를 clone후 자신만의 Repository를 만들어줍니다.
3. Vercel 홈페이지에서 NewProject 버튼을 누른 후 본인이 만든 Repository를 import해 줍니다. (저의 경우는 my_blog_template입니다.)

<img src="/static/images/make-dev-blog/import-repo.png" width="400" />

4. 팀 설정입니다. 다른 사람들과 협업하길 원한다면 설정해 줍니다.

<img src="/static/images/make-dev-blog/create-team.png" width="400" />

5. 프로젝트 설정입니다.

> PROJECT NAME : 프로젝트 이름입니다.  
> FRAMEWORK PRESET : 해당되는 프레임워크를 선택해 줍니다.  
> ROOT DIRECTORY : 프로젝트의 루트 디렉토리를 설정 할 수 있습니다.
>
> Build and Output Settings : build, output, install 시 사용할 터미널 커맨드를 설정 할 수 있습니다.  
> Environment Variables : 프로젝트 환경 변수를 정의해 줄 수 있습니다.

<img src="/static/images/make-dev-blog/configure-project.png" width="500" />

6. 파란색 Deploy 버튼을 누르면 자동으로 배포가 완료됩니다.
